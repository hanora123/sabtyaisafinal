# نظام نقطة البيع + المخزون + المحاسبة - منطق العمل ووحدات النظام

## منطق العمل (Core Business Logic)
معادلة المخزون:
- `Stock = Opening + Purchases - Sales - Transfers`

معادلة الربح:
- `Profit = Selling Price - Cost Price`

تدفق المحاسبة (قصد مثال):
- عند البيع:
  - قيد: مدين نقد/بنك، دائن إيراد
- عند انخفاض المخزون:
  - قيد: مدين COGS، دائن المخزون

## وحدات النظام المقترحة
خدمة POS:
- السلة، الإتمام، الإضافة السريعة، معالجة المدفوعات

خدمة المخزون:
- تتبع المخزون، التحويلات، تنبيهات نقص المخزون
- إدارة المخزون المحجوز

خدمة المحاسبة:
- توليد القيود المحاسبية للمبيعات/المشتريات/المرتجعات/المصاريف
- احتساب P&L والميزانية والتدفقات النقدية

خدمة التقارير:
- لوحات ومؤشرات (Analytics)
- تنبيهات وحركات مخزون

خدمة المستخدمين/المصادقة:
- تسجيل الدخول مع توجيه تلقائي حسب الدور (Admin -> لوحة الإدارة، Cashier -> نقطة البيع).
- فرض الصلاحيات (RBAC) مع دعم **أدوار متعددة** لكل مستخدم.
- واجهة إدارة شاملة للتحكم في حالة المستخدم (نشط/معطل) وتعديل أدواره.
- سجلات التدقيق (Audit Logs) لكل عمليات الدخول والخروج والعمليات الحساسة.
## أتمتة القيود (Double Entry)
قواعد عامة:
- كل حدث تجاري (بيع/شراء/مصروف/مرتجع) يجب أن ينتج عنه قيود متوازنة

مصادر القيود:
- قيود مصدر المبيعات
- قيود مصدر المشتريات
- قيود المصروفات
- تعديلات يدوية عند الحاجة
- قيود المدفوعات (دخول/خروج)
